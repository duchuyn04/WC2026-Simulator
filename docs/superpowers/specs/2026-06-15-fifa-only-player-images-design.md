# FIFA-Only Player Images Design Spec

- **Status**: Approved
- **Date**: 2026-06-15
- **Supersedes scope**: Image-source section of `2026-06-13-player-image-fallback-design.md` (the runtime fallback chain design still applies structurally, but `wikiPictureUrl` will no longer be populated by the data layer).

## 1. Context & Requirements

The current data layer writes player images to `data/fifa-teams-squads.json` from two sources:

1. FIFA API (`api.fifa.com` squad endpoint) — `pictureUrl` field.
2. Wikipedia/Wikidata — `wikiPictureUrl` field, populated by:
   - `fetch-fifa-teams-squads.mjs` (Wikipedia pageimage API for the squad's wikiTitle).
   - `enrich-wikidata-images.mjs` (Wikipedia pageimage → Wikidata Commons fallback for players still missing).

Current population (1248 players total):
- FIFA: 323
- Wikipedia: 795
- Wikipedia pageimage: 77
- Wikidata Commons: 1
- None: 52

**Goal:** Stop fetching and storing player images from Wikipedia/Wikidata. Use only the FIFA API for player images. The runtime fallback chain in `PortraitImage` (FIFA → fallback → placeholder) remains in code but `wikiPictureUrl` will no longer be populated, so the chain effectively collapses to FIFA → placeholder.

**Non-goals (out of scope):**
- Refactoring the frontend `fallbackSrc` prop away.
- Removing `wikiTitle`/`wikiName`/club fields from the data shape (still parsed from the Wikipedia squads page for club, caps, goals data).
- Removing the Wikipedia squads wikiText parse itself.
- Touching the coach image path (already FIFA-only via `PictureUrl`/`ThumbnailUrl` on the official object).

## 2. Proposed Changes

### A. `scripts/fetch-fifa-teams-squads.mjs`

Remove Wikipedia image fetching and write paths:

1. Delete the `fetchWikipediaImages` function (lines 282–312).
2. In `main()`:
   - Delete the `wikipediaImageTitles` accumulator and the `wikipediaImageTitles.push(wikiPlayer.wikiTitle)` call inside the `normalizedPlayers.map(...)` (lines 320, 337).
   - Delete the Wikipedia image write loop that sets `wikiPictureUrl` and overwrites `pictureSource` to `"wikipedia"` (lines 377–390).
3. Add a strip pass immediately before `writeFileSync` to clean pre-existing `wikiPictureUrl` and non-FIFA `pictureSource` values from the existing data (since the fetch overwrites player fields, but `wikiPictureUrl` was only ever added by the removed code; a one-time strip is still defensive against manual edits or future regressions):
   ```js
   let fifaCount = 0;
   let noneCount = 0;
   for (const team of enrichedTeams) {
     for (const player of team.squad) {
       if ("wikiPictureUrl" in player) delete player.wikiPictureUrl;
       if (player.pictureSource && player.pictureSource !== "fifa") {
         player.pictureSource = null;
       }
       if (player.pictureUrl) fifaCount += 1;
       else noneCount += 1;
     }
   }
   ```
4. Update the `enrichment` object: drop `wikipediaFallbackImages: wikipediaImageCount`. Add `imageSources: { fifa: <count>, none: <count> }` populated by the strip-pass counters. Drop `wikipediaTeams` if the count was only meaningful for image lookup — but keep it for now since it documents the Wikipedia squad page parse.

### B. `scripts/enrich-wikidata-images.mjs`

Delete the file. Its sole purpose is writing `wikiPictureUrl`; once that field is no longer maintained, the script has no effect.

### C. `package.json`

Remove the `"enrich:wikidata-images"` entry from `"scripts"`.

### D. Frontend

No code changes. The runtime fallback chain in `PortraitImage.tsx`, `PortraitLightbox.tsx`, and `TeamRoster.tsx` continues to accept `fallbackSrc={player.wikiPictureUrl}`. With `wikiPictureUrl` no longer populated, `fallbackSrc` is `undefined` and the chain collapses to FIFA → placeholder, which is the desired behavior.

The `Player` type in `TeamRoster.tsx` keeps the optional `wikiPictureUrl?: string | null` field for forward compatibility (no consumer code needs to change).

### E. Data file

`data/fifa-teams-squads.json` will be regenerated by the next `npm run fetch:teams-squads` run. Expected post-change stats:

- `enrichment.imageSources.fifa` = 323
- `enrichment.imageSources.none` = 925
- Total = 1248
- `wikiPictureUrl` occurrences = 0

`wikiTitle`, `wikiName`, and `club.wikiTitle` fields remain on players (and clubs) as inert data. They are no longer read by any image-lookup code path.

## 3. Testing and Verification Plan

1. **Unit tests:** Run `npm run test`. Per AGENTS.md, the test suite imports from `scripts/lib/tournament-stats.mjs`, not from `fetch-fifa-teams-squads.mjs` or `enrich-wikidata-images.mjs`. No test should fail. Grep for any test that references `wikiPictureUrl` and update/remove the assertion if found.
2. **Data regeneration:** Run `npm run fetch:teams-squads`. Confirm `data/fifa-teams-squads.json` updates without error.
3. **Data integrity checks** (run on the regenerated file):
   - `grep -c '"wikiPictureUrl"' data/fifa-teams-squads.json` → must equal 0.
   - `grep -c '"pictureSource": "wikipedia' data/fifa-teams-squads.json` → must equal 0.
   - `grep -c '"pictureSource": "wikidata-commons"' data/fifa-teams-squads.json` → must equal 0.
   - `grep -c '"pictureSource": "wikipedia-pageimage"' data/fifa-teams-squads.json` → must equal 0.
   - `grep -c '"pictureSource": "fifa"' data/fifa-teams-squads.json` → must equal 323.
   - `node -e "const d=require('./data/fifa-teams-squads.json'); console.log(d.enrichment.imageSources)"` → must show `{ fifa: 323, none: 925 }` (within ±1 player if FIFA squad data changed since last fetch).
4. **Build:** Run `npm run build` — must compile without TypeScript or ESLint errors.
5. **Browser verification:** Open team pages (Canada, USA, Croatia) and confirm:
   - Players with FIFA `pictureUrl` render the FIFA image.
   - Players without FIFA `pictureUrl` render the `PortraitPlaceholder` (initials + jersey number + "No image" badge).
   - Clicking a player tile opens the lightbox. If the FIFA URL 404s, the placeholder appears (no Wikipedia chain).

## 4. Rollback

A single `git revert <commit-sha>` undoes all source changes. To restore the prior `wikiPictureUrl` data, either:
- Re-run the reverted `npm run fetch:teams-squads` + `npm run enrich:wikidata-images`, OR
- Restore `data/fifa-teams-squads.json` from git history at the previous commit.
